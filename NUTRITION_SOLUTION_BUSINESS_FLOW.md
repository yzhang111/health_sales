# 营养解决方案辅助工具：业务流程设计方案

## 概要

本方案面向“营养素销售员”，目标是帮助销售员在合法合规前提下，完成客户信息采集、营养缺失评估、建议生成和跟进复评，提升服务质量与转化效率。

## 目标用户

- 营养素销售员

## 使用场景

### 案例 1：睡眠问题

- 客户主诉睡眠不好
- 销售员需要根据客户情况输出营养状况分析报告
- 提供可执行的营养补给建议与跟进计划

### 案例 2：术后营养补给

- 客户做完特定手术
- 销售员需要给出恢复期营养支持建议
- 输出分阶段补给建议与风险提示

## 核心功能

1. 记录客户基本信息、生活习惯和健康诉求
2. 分析客户可能存在的营养缺失情况
3. 生成面向客户的合法合规健康报告与销售话术

## 推荐 Skill 组合（按执行顺序）

1. `blueprint`
- 先把“建档-评估-建议-报告-跟进”的完整业务蓝图定清楚，避免后续功能碎片化。

2. `market-research`
- 建立营养建议依据和参考知识底座，用于不同人群、症状与术后阶段。

3. `api-design`
- 将业务流程落成标准化接口与状态流，便于后续系统实现。

4. `security-review`
- 处理健康数据权限、合规边界、免责声明、风险提示与转诊规则。

5. `prompt-optimizer`（可选）
- 若引入大模型生成报告/话术，用于提高提示词稳定性和输出一致性。

6. `verification-loop`（可选）
- 上线前做全链路验证，确保流程与合规模板可用。

## 业务流程设计（MVP）

1. 客户建档
- 采集：基础信息、生活方式、健康诉求、既往史、用药史、手术史、过敏与禁忌。

2. 风险分层
- 先做红线筛查（高风险症状、术后禁忌、药物冲突）。
- 命中红线时，系统明确提示“需医生评估”，暂停销售建议。

3. 营养缺失评估
- 基于规则引擎 + 问卷评分输出“可能缺失项”及置信度。
- 明确非诊断属性，不输出疾病诊断结论。

4. 方案生成
- 生成客户版报告：通俗解释、建议路径、执行周期、注意事项。
- 生成销售员版建议卡：沟通要点、补给节奏、复评节点、话术模板。

5. 合规包装
- 自动附加免责声明、适用边界、禁忌提醒与就医建议触发条件。

6. 跟进复评
- 按 2-4 周回访记录变化，更新建议并形成持续服务计划。

## 场景映射要点

### 场景 1：睡眠不好

- 重点采集：作息规律、压力水平、咖啡因摄入、运动情况、夜间觉醒频次。
- 输出：睡眠相关营养支持建议 + 生活方式建议 + 复评指标。

### 场景 2：术后补给

- 重点采集：手术类型、恢复阶段、医生嘱托、当前禁忌。
- 输出：分阶段补给建议（早期/恢复期）+ 风险提醒 + 转诊触发条件。

## 合规与风控边界

1. 系统结论定位为“营养支持建议”，非医疗诊断
2. 对高风险用户必须触发“先就医”规则
3. 健康数据按最小权限访问并记录操作审计
4. 所有对外报告默认带免责声明与禁忌提示

## 后续落地建议

1. 先确定最小可用字段集（建档/评估/报告/复评）
2. 再确定报告模板与销售话术模板
3. 最后进入接口设计与页面流程实现
